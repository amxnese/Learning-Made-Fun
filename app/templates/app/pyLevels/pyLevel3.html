Python's Built-in Data Structures

What are built-in Data Structures

Built-in data structures in Python are pre-defined types that allow for the storage and organization of data within a program. They are directly accessible without the need to import any external libraries, making them integral for everyday programming tasks. These data structures are optimized for performance and designed to provide a broad range of operations to manage collections of data efficiently. Each structure has unique characteristics and operations that make it suitable for specific kinds of tasks, like indexing, searching, updating, and removing data elements in a structured way.

Why we use built-in Data Structures

Built-in data structures in Python are fundamental components of the language that allow you to organize and manipulate data efficiently. Here are some reasons why we use them:

  • Efficiency: Built-in data structures like lists, tuples, dictionaries, and sets are highly optimized for performance. They are implemented in C under the hood, making operations like insertion, deletion, and lookup fast and efficient.
  • Versatility: Python's built-in data structures are versatile and can be used in a wide range of scenarios. Lists are flexible sequences of elements, dictionaries are key-value pairs for mapping, sets are unordered collections of unique elements, and tuples are immutable sequences.
  • Ease of Use: Python's built-in data structures are designed to be easy to use and understand. They come with built-in methods and functionalities that simplify common operations. For example, you can easily append an element to a list, iterate over a dictionary's keys or values, or perform set operations like union, intersection, and difference.
  • Expressiveness: Python's syntax for working with built-in data structures is concise and expressive. This makes code more readable and maintainable, as complex data manipulations can often be expressed in just a few lines of code.
  • Memory Management: Python's built-in data structures handle memory management automatically, making it easier for developers to focus on solving problems rather than worrying about memory allocation and deallocation.
  • Standardization: Built-in data structures are part of the Python standard library, which means they are available out of the box and can be used in any Python environment without the need for additional installations or dependencies.

Available Data Structures in Python

1. Lists:

Lists are ordered collections of items that can be of different data types. They are mutable, meaning their elements can be changed after creation.

Use Case: Lists are commonly used when you need a collection of items that you may need to modify, such as a list of tasks, names, or numbers. 

my_list = ['apple', 'banana', 'cherry']

2. Tuples:

Tuples are similar to lists, but they are immutable, meaning their elements cannot be changed after creation. They are often used to store related pieces of data together.

Use Case: Tuples are useful when you want to ensure that the data remains unchanged throughout the program, such as coordinates, RGB values, or data that is passed as arguments to functions. 

my_tuple = ('apple', 'banana', 'cherry')

3. Sets:

Sets are unordered collections of unique elements. They do not allow duplicate items. Sets are mutable, meaning you can add or remove items from them.

Use Case: Sets are handy for operations requiring uniqueness, such as removing duplicates from a list or checking for membership. 

my_set = {'apple', 'banana', 'cherry'}

4. Dictionaries:

Dictionaries are collections of key-value pairs. They are unordered, mutable, and can store any data type as values. Keys must be unique.

Use Case: Dictionaries are useful when you need to map keys to values, such as storing information about a person, product, or any entity with associated attributes. 

my_dict = {'name': 'John', 'age': 30, 'city': 'New York'}

Built-in methods Related To Built-in Datastrures

Lists

  • len(): Returns the number of items in a list.
  • append(): Adds an item to the end of a list.
  • pop(): Removes and returns the item at the specified index.
  • sort(): Sorts the items of a list in ascending order.

Examples:

my_list= [3, 1, 4, 1, 5, 9, 2, 6, 5]
print(len(my_list))  # Output: 9
          
my_list.append(7)
print(my_list)   # Output: [3, 1, 4, 1, 5, 9, 2, 6, 5, 7]
          
popped_item = my_list.pop(2)
print(popped_item) # Output: 4
print(my_list) # Output: [3, 1, 1, 5, 9, 2, 6, 5, 7]
          
my_list.sort()
print(my_list)# Output: [1, 1, 2, 3, 5, 5, 6, 7, 9]

Dictionaries

  • keys(): Returns a view object that displays a list of all the keys in the dictionary.
  • values(): Returns a view object that displays a list of all the values in the dictionary.
  • items(): Returns a view object that displays a list of tuple pairs (key, value) in the dictionary.
  • get(): Returns the value of the specified key. If the key does not exist, it returns None or the specified default value.

Examples:

my_dict = {'a': 1, 'b': 2, 'c': 3}
          
print(my_dict.keys())  # Output: dict_keys(['a', 'b', 'c'])
print(my_dict.values())  # Output: dict_values([1, 2, 3])
print(my_dict.items())  # Output: dict_items([('a', 1), ('b', 2), ('c', 3)])
          
print(my_dict.get('a'))  # Output: 1
print(my_dict.get('d', 'Key not found'))  # Output: Key not found

Sets

  • add(): Adds an element to the set.
  • remove(): Removes the specified element from the set. Raises KeyError if the element is not present.
  • union(): Returns a new set with elements from both sets.
  • intersection(): Returns a new set with elements that are common to both sets.

Examples:

my_set1 = {1, 2, 3}
my_set2 = {3, 4, 5}
          
my_set1.add(4)
print(my_set1)  # Output: {1, 2, 3, 4}
          
my_set1.remove(2)
print(my_set1)  # Output: {1, 3, 4}
          
union_set = my_set1.union(my_set2)
print(union_set)  # Output: {1, 3, 4, 5}
          
intersection_set = my_set1.intersection(my_set2)
print(intersection_set)  # Output: {3}
